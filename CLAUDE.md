## Project Overview

PyMO2NScr is a Haskell compiler that converts PyMO visual novel games to NScripter/ONScripter format. The project consists of a main executable and two Git submodules:
- `PyMO-Parser/`: Haskell library for parsing PyMO file formats (GameConfig, Script, etc.)
- `Nscripter-API/`: Reference documentation for NScripter commands

## Code Architecture

### Module Structure

1. **Main.hs** (src/Main.hs:1)
   - CLI interface with encoding support (GBK, UTF-8, ShiftJIS)
   - Parses command-line arguments and dispatches to compiler

2. **Compiler.hs** (src/Compiler.hs:1)
   - Core compiler monad (`Compiler` type using `RWST`)
   - Manages compilation state (variables, loaded scripts, compiled scripts)
   - Template-based output using `nscr-template.txt`
   - Key functions: `writeCompilerOutput`, `writeDefine`, `writeBody`

3. **ScriptCompiler.hs** (src/ScriptCompiler.hs:1)
   - Script-level compilation logic
   - Handles label splitting and block organization
   - Routes commands to CommandCompiler

4. **CommandCompiler.hs** (src/CommandCompiler.hs:1)
   - Implements translation of individual PyMO commands to NScripter
   - Contains command handlers for `goto`, `change`, `call`, etc.

### Data Flow

`Main.hs` → `Compiler.hs` → `ScriptCompiler.hs` → `CommandCompiler.hs`

The compiler uses a template system where `nscr-template.txt` contains placeholders (`&&header`, `&&defines`, `&&body`) that are replaced with generated content.

### Compiler State

The `CompilerState` (src/Compiler.hs:47) tracks:
- `csLocalVariables`: Local PyMO variables mapped to NScripter variables
- `csGlobalVariables`: Global PyMO variables mapped to NScripter variables
- `csLoadedScripts`: Cache of loaded PyMO scripts with their IDs
- `csCompiledScripts`: Set of already compiled script names

### Template System

Output is generated by filling holes in `src/nscr-template.txt`:
```
&&header

*define
&&defines
game

*start
&&body
```

The `runCompiler` function (src/Compiler.hs:157) replaces `&&header`, `&&defines`, and `&&body` with actual compiler output.

## Important Notes

- **Windows-specific paths**: Use absolute paths for executables as the system PATH may be incorrect
- **Stack environment**: Always set `STACK_ROOT=C:/sr` before using Stack commands
- **Encoding support**: The compiler handles multiple text encodings for different NScripter variants
- **Template-based output**: All NScr output is generated by filling holes in the template file

When modifying the compiler, ensure new output is written through the appropriate `write*` functions in `Compiler.hs` to maintain correct template formatting.
